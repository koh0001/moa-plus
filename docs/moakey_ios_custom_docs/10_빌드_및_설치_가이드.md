# Moakey iOS Custom - 빌드 및 설치 가이드

이 문서는 개발 완료된 코드를 Xcode에서 빌드하고 실제 아이폰에 설치하기까지의 전체 과정을 다룹니다.
신규 파일 16개가 추가되었고, App Group 설정이 필요하기 때문에 기존 빌드 환경에서도 아래 단계를 따라야 합니다.

---

## Part 1. Xcode 프로젝트 설정

### Step 1. 신규 파일을 Xcode 프로젝트에 추가

Xcode에서 `ios-moaki.xcodeproj`를 열고, 아래 파일들을 각 타겟에 추가합니다.

**방법**: 각 폴더를 우클릭 → `Add Files to "ios-moaki"...` → 파일 선택 → **Target Membership 체크 확인**

#### MoakiKeyboard 타겟에 추가할 파일 (9개)

| 폴더 | 파일 | 설명 |
|------|------|------|
| `MoakiKeyboard/Models/` | `SwipeProfile.swift` | 긋기 각도 프리셋 |
| | `ColumnGestureOverride.swift` | 열별 제스처 보정 |
| | `SecondaryKeyAction.swift` | 롱프레스 보조입력 매핑 |
| | `ShortcutExpansion.swift` | 약어 확장 데이터 모델 |
| | `ThemeSettings.swift` | 테마/외형 설정 |
| `MoakiKeyboard/Engine/` | `AbbreviationEngine.swift` | 약어 확장 엔진 |
| `MoakiKeyboard/Utilities/` | `GestureSettings.swift` | 통합 제스처 설정 |
| | `HapticManager.swift` | 이벤트별 햅틱 관리 |
| | `BackgroundImageManager.swift` | 배경 이미지 관리 |

#### MoakiKeyboard 타겟에 추가할 파일 (4개 - 뷰/뷰모델)

| 폴더 | 파일 | 설명 |
|------|------|------|
| `MoakiKeyboard/ViewModels/` | `KeyboardViewModel.swift` | 키보드 뷰모델 (기존에서 분리됨) |
| `MoakiKeyboard/Views/` | `SpecialCharacterLayerView.swift` | 특수문자 레이어 |
| | `AbbreviationCandidateView.swift` | 약어 후보 바 |

> `ViewModels/` 폴더가 새로 생겼으므로 Xcode에서 Group을 먼저 만들어야 할 수 있습니다.

#### ios-moaki (메인 앱) 타겟에 추가할 파일 (7개)

| 폴더 | 파일 | 설명 |
|------|------|------|
| `ios-moaki/Settings/` | `SettingsMainView.swift` | 설정 메인 화면 |
| | `InputSettingsView.swift` | 모아키 입력 설정 |
| | `SecondaryInputSettingsView.swift` | 보조 입력 설정 |
| | `SpecialCharSettingsView.swift` | 특수문자 설정 |
| | `AbbreviationSettingsView.swift` | 단축어/약어 설정 |
| | `AppearanceSettingsView.swift` | 외형 설정 |
| | `FeedbackSettingsView.swift` | 반응 설정 |

#### 공유 타겟 멤버십이 필요한 파일

설정 앱에서 키보드 모델 타입을 참조하므로, 아래 Model 파일들은 **두 타겟 모두**에 체크합니다:

- `SwipeProfile.swift`
- `ColumnGestureOverride.swift`
- `SecondaryKeyAction.swift`
- `ShortcutExpansion.swift`
- `ThemeSettings.swift`
- `GestureSettings.swift`
- `KeyboardSettings.swift`
- `SpecialCharacterLayerView.swift` (`SpecialCharCategory` enum 때문)

**방법**: 파일 선택 → 우측 File Inspector → Target Membership에서 `ios-moaki`, `MoakiKeyboard` 둘 다 체크

### Step 2. App Group 설정

키보드 익스텐션과 메인 앱이 설정값을 공유하려면 App Group이 필요합니다.

#### 2-1. ios-moaki (메인 앱) 타겟

1. 프로젝트 네비게이터에서 프로젝트 파일 클릭
2. **TARGETS** → `ios-moaki` 선택
3. **Signing & Capabilities** 탭
4. **+ Capability** 클릭 → `App Groups` 검색 후 추가
5. **+** 버튼으로 그룹 추가: `group.com.moaki.keyboard`

#### 2-2. MoakiKeyboard (키보드 익스텐션) 타겟

1. **TARGETS** → `MoakiKeyboard` 선택
2. 동일하게 **Signing & Capabilities** → **+ Capability** → `App Groups`
3. 동일한 그룹 추가: `group.com.moaki.keyboard`

> 두 타겟에 동일한 App Group ID가 있어야 `UserDefaults(suiteName:)`로 데이터를 공유할 수 있습니다.

### Step 3. 전체 접근 허용 (Open Access) 설정

App Group 공유 컨테이너에 접근하려면 키보드 익스텐션에 Open Access가 필요합니다.

`MoakiKeyboard/Info.plist`에서 `RequestsOpenAccess`를 `true`로 변경:

```xml
<key>RequestsOpenAccess</key>
<true/>
```

> 현재 `false`로 되어 있습니다. Xcode의 Info.plist 에디터에서 직접 변경하거나, 소스 코드로 열어서 수정하세요.

### Step 4. 서명 설정

1. **TARGETS** → `ios-moaki` → **Signing & Capabilities**
2. **Team**: 본인의 Apple Developer 계정 선택
3. **Bundle Identifier**: 고유값 확인 (예: `com.yourname.ios-moaki`)
4. **TARGETS** → `MoakiKeyboard` → 동일하게 서명 설정
5. **Bundle Identifier**: 메인 앱의 하위 (예: `com.yourname.ios-moaki.MoakiKeyboard`)

> 무료 Apple ID도 가능하지만, App Group은 유료 개발자 계정이 필요할 수 있습니다.
> 무료 계정으로 테스트할 경우 App Group 없이도 기본 키보드 기능은 동작합니다 (설정 공유만 안 됨).

### Step 5. 빌드 확인

1. 상단 바에서 빌드 대상을 **실기기** 또는 **시뮬레이터**로 선택
2. `Cmd + B` → 빌드 성공 확인
3. 에러 발생 시:
   - "Cannot find type in scope" → Step 1의 타겟 멤버십 확인
   - "Provisioning profile" → Step 4의 서명 설정 확인
   - "App Group" 관련 → Step 2 재확인

## Part 2. 아이폰 기기 설정

### Step 6. 개발자 모드 활성화 (iOS 16+)

iOS 16부터 실기기에 개발 빌드를 설치하려면 개발자 모드가 필요합니다.

1. 아이폰을 **USB 케이블로 Mac에 연결**
2. Xcode에서 기기가 인식되는지 확인 (Window → Devices and Simulators)
3. 아이폰에서: **설정** → **개인정보 보호 및 보안** → **개발자 모드**
4. 토글을 **켜기**
5. **재시작** 팝업에서 확인
6. 재부팅 후 잠금 해제 → **"개발자 모드를 켜겠습니까?"** 팝업에서 **켜기**

> "개발자 모드" 메뉴가 보이지 않으면: Xcode에서 기기를 한 번 인식시킨 후 다시 확인하세요.

### Step 7. 앱 설치 (빌드 & 실행)

1. Xcode 상단 바에서 빌드 대상을 **연결된 아이폰**으로 선택
2. `Cmd + R` (Run) → 빌드 후 아이폰에 자동 설치
3. 처음 설치 시 아이폰에 **"신뢰할 수 없는 개발자"** 경고가 나올 수 있음

### Step 8. 개발자 인증서 신뢰 (첫 설치 시)

"신뢰할 수 없는 개발자" 메시지가 나타나면:

1. **설정** → **일반** → **VPN 및 기기 관리**
2. **개발자 앱** 섹션에서 본인의 Apple ID 탭
3. **"[Apple ID]" 신뢰** → 확인

### Step 9. 키보드 추가

1. **설정** → **일반** → **키보드** → **키보드** → **새 키보드 추가...**
2. **타사 키보드** 섹션에서 **MoakiKeyboard** 선택
3. 목록에 MoakiKeyboard가 추가된 것을 확인

### Step 10. 전체 접근 허용

App Group 기반 설정 공유를 위해 필요합니다:

1. **설정** → **일반** → **키보드** → **키보드**
2. **MoakiKeyboard** 탭
3. **전체 접근 허용** 토글 켜기
4. 경고 팝업에서 **허용**

> 전체 접근을 허용하지 않아도 기본 입력은 동작합니다. 설정 앱과 키보드 간 설정 동기화만 안 됩니다.

### Step 11. 키보드 전환 및 사용

1. **메모**, **메시지** 등 텍스트 입력이 가능한 앱 열기
2. 키보드 하단의 **🌐 버튼**을 **길게 누르기**
3. 목록에서 **MoakiKeyboard** 선택
4. 모아키 키보드가 표시되면 설치 완료

## 빠른 체크리스트

```
Xcode:
  [ ] 신규 16개 파일 프로젝트에 추가 + 타겟 멤버십 확인
  [ ] 공유 모델 파일 7+1개 양쪽 타겟 체크
  [ ] App Group 추가: group.com.moaki.keyboard (양쪽 타겟)
  [ ] Info.plist: RequestsOpenAccess = true
  [ ] 서명 설정 (Team + Bundle ID)
  [ ] Cmd+B 빌드 성공 확인

iPhone:
  [ ] 개발자 모드 활성화 (설정 → 개인정보 보호 및 보안)
  [ ] Cmd+R 으로 기기에 설치
  [ ] 개발자 인증서 신뢰 (첫 설치 시)
  [ ] 키보드 추가 (설정 → 일반 → 키보드)
  [ ] 전체 접근 허용
  [ ] 🌐 길게 눌러 키보드 전환
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 키보드 목록에 MoakiKeyboard가 없음 | 앱이 설치되지 않았거나 익스텐션 빌드 실패 | Xcode에서 MoakiKeyboard 스킴 빌드 확인 |
| 키보드 전환 후 빈 화면 | KeyboardViewController 로드 실패 | Xcode 콘솔에서 에러 확인, 메모리 초과 여부 점검 |
| 설정 앱에서 변경해도 키보드에 반영 안 됨 | App Group 미설정 또는 전체 접근 미허용 | Step 2, Step 3, Step 10 재확인 |
| "신뢰할 수 없는 개발자" 무한 반복 | 인증서 만료 (무료 계정은 7일) | Xcode에서 다시 빌드 → 자동 갱신 |
| 빌드 시 "Provisioning profile" 에러 | 서명 설정 불일치 | Step 4에서 Team/Bundle ID 재설정 |
| 키보드에서 햅틱이 안 됨 | 시뮬레이터에서는 햅틱 미지원 | 실기기에서 테스트 |

> 작성일: 2026-03-27

---

## 1. 프로젝트 개요

### 목표
갤럭시 원본 양손 모아키 사용감을 iOS 커스텀 키보드로 최대한 복원하고, iOS 제약 안에서 생산성 기능을 확장한다.

### 핵심 원칙
1. 입력 정확도 우선
2. 원본 모아키 감각 유지
3. 설정은 많아도 단계별로 노출
4. iOS 제약은 우회하되 사용감은 삼성 키보드에 가깝게

### 2.4 핵심 코드 파일별 역할

| 파일 | 핵심 역할 | 수정 시 영향 범위 |
|------|----------|-----------------|
| `HangulComposer.swift` | 한글 조합 상태머신 (empty→choseong→choseongJungseong→complete) | 모든 텍스트 입력에 영향 |
| `GestureAnalyzer.swift` | 터치 포인트 시퀀스 → 방향 배열 변환. threshold 20px, reversalThreshold 10px | 모음 입력 정확도에 직접 영향 |
| `VowelResolver.swift` | 방향 배열 → Jungseong 변환. PatternTrie 사용 | 모음 인식에 직접 영향 |
| `KeyboardViewModel` | (KeyboardView.swift 내부) 입력 흐름 총괄. 제스처→조합→출력 연결 | UI와 엔진 사이 허브 |
| `KeyboardMetrics.swift` | 그리드 레이아웃, 키 크기, 숫자 매핑 정의 | 레이아웃 변경 시 핵심 |
| `KeyboardSettings.swift` | 현재 `showGesturePreview` 1개만 관리 | 설정 시스템 확장 시 대폭 변경 필요 |
| `KeyboardViewController.swift` | UIKit ↔ SwiftUI 브릿지, 텍스트 프록시 연동 | 시스템 키보드 전환, 텍스트 입출력 |

## 6. 마일스톤 기반 개발 로드맵

### M1: 입력 엔진 기반 (핵심 정확도)

> **목표**: 원본 갤럭시 모아키와 동일한 입력 정확도 달성

#### M1-1. 인프라 정비

| 작업 | 관련 파일 | 설명 |
|------|----------|------|
| ViewModel 분리 | `KeyboardView.swift` → `KeyboardViewModel.swift` (신규) | 401줄 파일에서 ViewModel을 별도 파일로 추출 |
| App Group 설정 | Xcode 프로젝트 설정 | 앱 + 익스텐션 공유 컨테이너 생성 |
| SharedDefaults 구축 | `KeyboardSettings.swift` → 구조체 기반 리팩터 | `UserDefaults(suiteName:)` 전환 |
| 설정 모델 정의 | `GestureSettings.swift` (신규) | `SwipeProfile`, `ColumnGestureOverride` 모델 |

#### M1-2. 레이아웃 복원

| 작업 | 관련 파일 | 설명 |
|------|----------|------|
| 키 타입 확장 | `KeyboardMetrics.swift` | `KeyContent`에 `vowelPrimitive`, `functional`, `systemSwitch` 추가 |
| 그리드 재설계 | `KeyboardMetrics.swift` | 원본 갤럭시 기준 좌측 기능열 + 중앙 자음 + 우측 모음 프리미티브 |
| 모음 프리미티브 키 | `KeyView.swift` | ㆍ/ㅣ/ㅡ 키 시각 분리 + 엔진 연결 |
| 하단열 재설계 | `FunctionRowView.swift` | 모아키 기능 우선 + iOS 필수키 최소 통합 |

#### M1-3. 제스처 엔진 고도화

| 작업 | 관련 파일 | 설명 |
|------|----------|------|
| 긋기 각도 프리셋 | `GestureDirection.swift`, `SwipeProfile` (신규) | 양손/오른손/왼손/직접설정 4종 |
| 긋기 길이 enum | `GestureAnalyzer.swift` | short/normal/long → pt 매핑 |
| 열별 제스처 보정 | `ColumnGestureOverride` (신규) | rotationOffsetDeg, verticalIWidthDelta 등 |
| 끝열 바깥 긋기 강화 | `ColumnGestureOverride` | outwardDistanceMultiplier 좌/우 끝열 별도 |
| 판정 우선순위 엔진 | `GestureAnalyzer.swift` | 탭→거리→긋기/롱프레스 분기 상태머신 |

#### M1 테스트 계획

| 테스트 | 대상 |
|--------|------|
| 열별 보정 정확도 테스트 | 1~5열 각 키의 8방향 긋기 판정 정확도 |
| 프리셋별 각도 분류 테스트 | 양손/오른손/왼손/직접설정 시나리오 |

| 탭/스와이프/롱프레스 충돌 테스트 | 빠른 이동 → 긋기, 정지 유지 → 롱프레스 |
| 레이아웃 회귀 테스트 | 기존 21개 모음 패턴 전수 동작 검증 |

## 8. 데이터 모델 설계

### 8.1 제스처 설정

```swift
// SwipeProfile - 긋기 각도 프리셋
struct SwipeProfile {
    enum Mode: String, Codable { case right, left, both, custom }
    var mode: Mode = .both
    var sectorAngles: [Double] = Array(repeating: 45.0, count: 8)
    var minSwipeDistance: SwipeLength = .normal
}

enum SwipeLength: String, Codable {
    case short, normal, long
    var threshold: CGFloat {
        switch self {
        case .short:  return 12.0
        case .normal: return 20.0
        case .long:   return 30.0
        }
    }
}

// ColumnGestureOverride - 열별 보정
struct ColumnGestureOverride: Codable {
    var columnId: Int                        // 1...5
    var rotationOffsetDeg: Double = 0        // 전체 각도 회전 보정
    var verticalIWidthDelta: Double = 0      // ㅣ 인식 폭 확장
    var horizontalEuWidthDelta: Double = 0   // ㅡ 인식 폭 확장
    var outwardDistanceMultiplier: Double = 1.0  // 바깥쪽 거리 가중
}
```

### 8.2 보조 입력

```swift
// SecondaryKeyAction - 키별 롱프레스 매핑
struct SecondaryKeyAction: Codable {
    var keyId: String                // 키 식별자 (예: "ㅂ")
    var visibleHint: String          // 작은 힌트 텍스트 (예: "1")
    var primaryLongPressOutput: String  // 롱프레스 기본 출력
    var popupOutputs: [String]       // 팝업 후보 목록
}
```

### 8.3 약어 확장

```swift
// ShortcutExpansion - 약어 데이터
struct ShortcutExpansion: Codable, Identifiable {
    var id: UUID = UUID()
    var trigger: String              // 트리거 (예: "ㅎㅅㅁㅇ")
    var replacement: String          // 치환 결과 (예: "koh@move.kr")
    var autoCommitMode: CommitMode = .onDelimiter
    var isEnabled: Bool = true

    enum CommitMode: String, Codable {
        case suggestion    // 후보 표시 후 탭
        case onDelimiter   // delimiter 입력 시 자동 확정
    }
}
```

### 8.4 테마 설정

```swift
// ThemeSettings - 외형 설정
struct ThemeSettings: Codable {
    var appearanceMode: AppearanceMode = .system
    var buttonThemeId: String = "default-gray"
    var backgroundImageId: String? = nil
    var backgroundOpacity: Double = 0.3
    var hapticEnabled: Bool = true
    var hapticStrength: HapticStrength = .normal
    var clickSoundEnabled: Bool = false

    enum AppearanceMode: String, Codable { case system, light, dark }
    enum HapticStrength: String, Codable { case light, normal }
}
```

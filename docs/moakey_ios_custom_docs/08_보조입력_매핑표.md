# 보조입력 매핑표

## 문서 목적
이 문서는 키 롱프레스 즉시 보조입력, 작은 힌트 표시, 확장 특수문자 레이어 구성, 약어 확장 예시를 한 번에 정리한 작업용 매핑 문서다.

## 1. 설계 원칙
- 롱프레스 즉시 보조입력은 `빠른 접근성`이 목적이다.
- 많은 특수문자는 별도 특수문자 레이어에서 제공한다.
- 작은 힌트는 숫자/대표 기호만 보여주고, 전체 목록은 팝업에 맡긴다.
- 약어 확장은 별도 엔진으로 처리한다.

## 2. 1차 숫자 매핑안
아래 매핑은 사용자가 빠르게 외우기 쉬운 `좌 -> 우` 순서를 기준으로 한 초안이다.

| 키 | 힌트 | 롱프레스 기본 출력 | 팝업 후보 예시 |
|---|---:|---|---|
| ㅃ | 1 | 1 | 1, !, ~ |
| ㅉ | 2 | 2 | 2, @ |
| ㄸ | 3 | 3 | 3, # |
| ㄲ | 4 | 4 | 4, $ |
| ㅆ | 5 | 5 | 5, % |
| ㅂ | 6 | 6 | 6, ^ |
| ㅈ | 7 | 7 | 7, & |
| ㄷ | 8 | 8 | 8, * |
| ㄱ | 9 | 9 | 9, (, ) |
| ㅅ | 0 | 0 | 0, -, _, +, = |

> 주의: 이 표는 1차 매핑 초안이다. 원본 삼성 키보드/모아키의 실제 롱프레스 매핑 캡처를 확보하면 재조정해야 한다.

## 3. 작은 힌트 표시 규칙

### 공통 규칙
- 메인 자음보다 확실히 작은 크기
- 키 우상단 기준 배치
- opacity는 메인 라벨보다 낮게

### 끝열 규칙
- 왼쪽 끝열: 힌트를 오른쪽 안쪽으로 당김
- 오른쪽 끝열: 힌트를 왼쪽 안쪽으로 당김
- 필요 시 `작게 / 보통 / 크게` 옵션 제공

## 4. 롱프레스 팝업 규칙

### 기본 규칙
- 길게 누르면 대표 숫자/기호 팝업 오픈
- 드래그로 후보 선택
- 손을 떼면 확정

### 충돌 처리
- 빠르게 움직이면 swipe 우선
- 일정 거리 이하 유지 시 longPress 후보 유지
- move 발생 시 longPress 예약 취소 가능

## 5. 확장 특수문자 레이어
많은 특수문자는 별도 레이어에서 제공한다.

### 진입 방식
- 언어 변환 계열 키 `짧게 탭`: 내부 특수문자 레이어
- 같은 키 `길게 누름`: 시스템 키보드 전환

### 1페이지 제안
| 그룹 | 문자 |
|---|---|
| 숫자 | 1 2 3 4 5 6 7 8 9 0 |
| 기본 문장부호 | ! ? . , : ; |
| 계정/웹 | @ # % & |
| 연산/기본 | + - = _ |
| 경로/개발 | / \\ | ~ |
| 괄호 | ( ) [ ] |

### 2페이지 제안
| 그룹 | 문자 |
|---|---|
| 중괄호/꺾쇠 | { } < > |
| 따옴표 | ' " ` |
| 화살표 | → ← ↑ ↓ |
| 문서 기호 | • ○ ● ※ |
| 개발 확장 | ; : | _ ~ ^ |

## 6. 약어 확장 예시 표

| 트리거 | 치환 결과 | 용도 |
|---|---|---|
| ㅎㅅㅁㅇ | koh@move.kr | 이메일 |
| ㅈㅅㅎㄴㄷ | 죄송합니다. 확인 후 다시 회신드리겠습니다. | 회신 문구 |
| ㅎㅇㅎㅅ | 확인했습니다. | 업무 답변 |
| ㅁㅂㅁㅈ | move branch merge 진행 | 개발 메모 |
| ㅂㄹㄱㅎㅅ | #AI #테크 #투자 | 블로그 해시태그 |

## 7. 약어 확정 규칙

### 권장 플로우
1. 사용자가 트리거 문자열 입력
2. 상단 후보 영역에 치환 후보 노출
3. 스페이스/엔터/문장부호 입력 시 확정
4. 치환 직후 backspace 1회로 트리거 원문 복원

### 정책
- 정확히 일치할 때만 동작
- 조합 중인 한글은 완성 후 검사
- 일반 입력을 방해하지 않도록 후보 기반 확정이 우선

## 8. 추후 확장 가능 항목
- 키별 팝업 후보 순서 편집
- 최근 사용 보조문자 우선 정렬
- 앱별 단축어 세트
- 업무용 / 개인용 프로파일 분리

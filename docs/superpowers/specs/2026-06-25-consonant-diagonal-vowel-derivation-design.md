# 설계 — 자음 대각선 ㅣ/ㅡ 진입 후 천지인 파생모음

> 작성 2026-06-25. brainstorming 승인(매핑표 확정). 다음: writing-plans.

## 1. 배경 / 문제

자음 키에서 대각선으로 ㅣ/ㅡ를 시작한 뒤(↗↖=ㅣ, ↙↘=ㅡ) 이어 그으면 **ㅢ 외에 아무 파생모음도 나오지 않는다**.

원인 (확인됨):
- `VowelResolver.resolve`는 첫 획이 대각선이면 `normalizeFirstStroke`가 `.upRight`/`.downRight`로 정규화한 뒤 `PatternTrie.match`를 탄다.
- trie의 **대각선 시작 파생 패턴은 ㅢ(`↘↖`/`↘↑`)뿐**이다 (`VowelPattern.swift:42-43`). ㅘ·ㅚ·ㅝ·ㅐ·ㅔ 등 복합모음은 전부 카디널(↑↓←→) 시작 패턴(ㅘ=↑→, ㅚ=↑↓ 등).
- 따라서 ↗(ㅣ) 시작은 후속과 무관하게 ㅣ, ↙↘(ㅡ) 시작은 위쪽(↖/↑) 후속만 ㅢ, 나머지는 ㅡ.

## 2. 목표 / 비목표

**목표**: 자음 대각선 진입(↗↖=ㅣ, ↙↘=ㅡ) 후 후속 방향으로 21개 모음 전부를 천지인 정통 규칙대로 파생.

**비목표 (YAGNI)**:
- 진입 좌/우 구별(↗≠↖) — 천지인은 구별하지 않음. ↗=↖=ㅣ, ↙=↘=ㅡ.
- 사용자 원래 11개 명세의 비대칭 케이스(↗←=ㅐ 등) — 천지인 정통으로 대체(brainstorming 합의).
- 전용 ㅣ/ㅡ 모음키, 자음 카디널 경로, 천지인 누적 변경.

## 3. 확정 매핑 (천지인 정통)

이 매핑은 **전용 ㅣ/ㅡ 모음키의 규칙과 동일**하다.

**ㅣ 시작 (↗ 또는 ↖):**
| 후속 | 결과 | 후속 | 결과 |
|---|---|---|---|
| (없음) | ㅣ | →← | ㅐ |
| → | ㅏ | ←→ | ㅔ |
| ← | ㅓ | ↓→ | ㅒ |
| ↑ | ㅕ | ↑→ | ㅖ |
| ↓ | ㅑ | | |

**ㅡ 시작 (↙ 또는 ↘):**
| 후속 | 결과 | 후속 | 결과 |
|---|---|---|---|
| (없음) | ㅡ | ↑→ | ㅘ |
| ↑ | ㅗ | ↑← | ㅚ |
| ↓ | ㅜ | ↓← | ㅝ |
| ← | ㅛ | ↓→ | ㅟ |
| → | ㅠ | ↑→← | ㅙ |
| ↗/↖(ㅣ) | ㅢ | ↓←→ | ㅞ |

## 4. 설계

### 4.1 핵심 아이디어
자음 키 제스처에서 **첫 획이 대각선(↗↖↙↘)이면 그것을 ㅣ/ㅡ primitive 진입으로 보고, 나머지 후속 방향을 전용 ㅣ/ㅡ 키와 같은 로직으로 해석**한다. 매핑이 동일하므로 전용키의 멀티스트로크 경로(`resolveVowelFromPrimitiveDrag` / `baseVowelFor` / `applySecondaryStroke`)를 재사용한다.

### 4.2 적용 지점
- `KeyboardViewModel.handleKoreanModeGesture`의 자음 키 분기 (`gestureEnded`): `directions[0]`이 대각선이면 primitive(↗↖→`.bar`(ㅣ), ↙↘→`.dash`(ㅡ))로 변환하고 후속 `directions[1...]`를 전용키 로직으로 처리. 후속이 없으면 현행대로 단독 ㅣ/ㅡ.
- 실시간 미리보기(`gestureMoved`의 자음 키 `peekVowel`)도 동일 분기.
- **slot-B 모음 키 / 자음 카디널 시작 / 전용 ㅣ/ㅡ 키는 무변경.**

### 4.3 구현 선택지 (plan에서 확정)
(a) 자음 키 분기에서 대각선 첫 획을 primitive로 변환 후 기존 전용키 함수 재사용 — 코드 재사용 최대, 추천.
(b) trie에 대각선 시작 파생 패턴을 추가 — `normalizeFirstStroke`가 대각선을 카디널로 보내는 현 구조와 충돌하므로 비추천.

## 5. 회귀 안전성

- **자음 카디널 시작 무변경**: →=ㅏ, ↑=ㅗ, ↑→=ㅘ 등 기존 trie 경로 그대로. (자음 대각선 ↗→=ㅏ와 결과가 겹치지만 무해 — 둘 다 ㅏ)
- **자음 대각선 단독 무변경**: ↗=ㅣ, ↘=ㅡ.
- **전용 ㅣ/ㅡ 키 무변경**: `resolveVowelFromPrimitiveDrag` 재사용하되 호출부만 추가.
- 기존 `VowelResolverTests`, `KeyboardViewModelVowelDragTests`, `GestureAnalyzerTests` 전부 통과 유지.
- 직전 skip fix(`PatternTrie.match` continue)와 공존.

## 6. 엣지케이스 / 리스크

- **후속 대각선(ㅢ)**: ㅡ 진입 후 ㅣ방향(↗/↖) = ㅢ. 자음 키는 `forceCardinalOnly=false`라 대각선 후속을 받을 수 있음(전용키는 4방향 한정과 대조). `resolveVowelFromPrimitiveDrag`/`baseVowelFor`가 대각선 후속을 처리하는지 plan에서 확인, 없으면 ㅢ 경로 보강.
- **자음 카디널 vs 대각선 결과 중복**: ↗→=ㅏ = →=ㅏ. 의도된 무해 중복.
- **false-positive**: 대각선 시작 의도가 아닌데 첫 획이 대각선으로 잡히면 파생 경로로 빠질 수 있음 → 실기기 QA.
- **GestureAnalyzer 첫 획 인식**: 대각선↔카디널 전환 인식 정확도(직전 skip fix가 일부 흡수).

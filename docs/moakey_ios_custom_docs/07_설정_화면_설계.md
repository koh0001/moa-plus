# 설정 화면 설계

## 문서 목적
이 문서는 모아키 iOS 커스텀 키보드의 설정 정보 구조(IA), 사용자 노출 우선순위, 각 설정 화면의 항목 구성을 정리한 설계 문서다.

## 설계 원칙
1. 자주 쓰는 설정부터 보이게 한다.
2. 엔진에 큰 영향을 주는 항목은 명확한 이름으로 분리한다.
3. 고급 설정은 접어두되 숨기지는 않는다.
4. 원본 모아키 느낌과 iOS 사용성 사이에서 조작 피로도를 최소화한다.

## 1. 최상위 설정 메뉴 구조

### 1) 모아키 입력
- 긋기 각도
- 긋기 길이
- 세로 라인별 제스처 보정

### 2) 보조 입력
- 키 내부 작은 힌트 표시
- 힌트 크기
- 힌트 위치
- 롱프레스 보조 입력 편집

### 3) 특수문자
- 내부 특수문자 레이어 진입 방식
- 특수문자 탭 구성
- 개발용 기호 확장 여부

### 4) 단축어 / 약어 확장
- 단축어 추가
- 후보 표시 방식
- delimiter 확정
- backspace 복원

### 5) 외형
- 시스템 / 라이트 / 다크
- 버튼 색상 테마
- 배경 이미지
- 배경 투명도

### 6) 반응
- 키 클릭 사운드
- 햅틱 on/off
- 햅틱 강도

## 2. 긋기 각도 화면

### 기본 항목
- 오른손용
- 왼손용
- 양손용
- 직접 설정

### 화면 설명 문구 예시
- 오른손용: 오른손 위주 사용 습관에 맞는 프리셋
- 왼손용: 왼손 위주 사용 습관에 맞는 프리셋
- 양손용: 좌우 균형형 45도 프리셋
- 직접 설정: 세부 각도 직접 조정

### 하단 보조 액션
- 테스트 모드 진입
- 기본값으로 복원

## 3. 긋기 길이 화면

### 항목
- 짧게
- 보통
- 길게

### 설명 문구
- 짧게: 민감하게 반응
- 보통: 기본값
- 길게: 더 크게 긋는 사용자용

## 4. 세로 라인별 제스처 보정 화면

### 목록형 화면
- 1열 `ㅃ / ㅂ / ㅁ / ㅋ`
- 2열 `ㅉ / ㅈ / ㄴ / ㅌ`
- 3열 `ㄸ / ㄷ / ㅇ / ㅊ`
- 4열 `ㄲ / ㄱ / ㄹ / ㅍ`
- 5열 `ㅆ / ㅅ / ㅎ`

### 상세 화면 항목
- 회전 보정
- `ㅣ` 인식 폭 보정
- `ㅡ` 인식 폭 보정
- 바깥쪽 긋기 민감도
- 테스트 버튼
- 해당 열만 기본값 복원

### 정책
- 3열은 기본적으로 전역값 사용
- 좌우 끝열은 outward 보정 항목 노출
- 고급 사용자만 수정하도록 별도 설명 제공

## 5. 보조 입력 설정 화면

### 5.1 기본 표시 옵션
- 보조 힌트 표시: 켜기 / 끄기
- 보조 힌트 크기: 작게 / 보통 / 크게
- 보조 힌트 위치: 기본 / 안쪽

### 5.2 롱프레스 보조 입력 편집
- 키별 대표 숫자/기호 변경
- 롱프레스 팝업 문자 목록 편집
- 기본 매핑 복원

### UX 원칙
- 사용자는 전체 키를 일괄 편집하기보다 개별 키 진입으로 수정할 수 있어야 한다.
- 끝열 키는 별도 미리보기 제공이 유용하다.

## 6. 특수문자 설정 화면

### 진입 규칙
- 언어 변환 계열 키 짧게 탭: 내부 특수문자 레이어
- 길게 누름: 시스템 키보드 전환

### 설정 항목
- 특수문자 1페이지 구성
- 특수문자 2페이지/카테고리 사용 여부
- 개발용 기호 우선 노출
- 최근 사용 문자 우선 표시 여부

## 7. 단축어 / 약어 확장 설정 화면

### 목록 화면
- 단축어 리스트
- 검색
- 즐겨찾기 필터
- 추가 버튼

### 상세 편집 화면
- 트리거 문자열
- 치환 문자열
- 확정 방식
  - 후보 표시 후 탭
  - delimiter 입력 시 확정
- 활성화 여부

### 부가 옵션
- 후보 표시 위치
- 자동 띄어쓰기 여부
- 치환 직후 backspace 복원 사용 여부

## 8. 외형 설정 화면

### 8.1 테마 모드
- 시스템
- 라이트
- 다크

### 8.2 버튼 색상 테마
- 기본 그레이
- 삼성 느낌 다크
- 블루 포인트
- 네이비
- 베이지
- 커스텀(후순위)

### 8.3 배경 이미지
- 없음
- 기본 이미지 선택
- 사용자 이미지 선택

### 8.4 배경 투명도
- 0% ~ 60% 권장
- 미리보기 제공

### 정책
- 배경은 이미지 + 투명도 정도로 제한
- 키 자체는 반투명 카드형 유지
- 가독성 보정 오버레이는 자동 적용 가능해야 함

## 9. 반응 설정 화면

### 항목
- 키 클릭 사운드
- 햅틱 반응
- 햅틱 강도
  - 약하게
  - 보통
- 이벤트별 햅틱 사용
  - 탭
  - 롱프레스 팝업 진입
  - 레이어 전환
  - 약어 확정

### 정책
- move 이벤트 연속 햅틱은 비권장
- 확정 순간 중심으로만 진동을 준다.

## 10. 설정 저장 구조
키보드 확장과 설정 앱은 App Group 기반 공유 저장소를 사용한다.

### 저장 대상
- 제스처 설정
- 세로 라인별 보정값
- 보조 입력 매핑
- 단축어/약어 확장 데이터
- 테마/배경 설정
- 반응 설정

## 11. 단계별 노출 정책

### 초급 사용자에게 우선 노출
- 긋기 각도
- 긋기 길이
- 보조 힌트 크기
- 다크/라이트/시스템
- 햅틱 on/off

### 고급 설정으로 숨김
- 세로 라인별 제스처 보정
- 키별 롱프레스 편집
- 특수문자 페이지 편집
- 약어 확정 방식 세부 옵션
- 배경 이미지 세부 투명도

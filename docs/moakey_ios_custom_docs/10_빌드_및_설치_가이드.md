# Moakey iOS Custom - 빌드 및 설치 가이드

Xcode를 처음 사용하는 분을 위한 단계별 가이드입니다.
각 단계에서 "어디를 클릭하는지"까지 설명합니다.

---

## 시작하기 전에

### Xcode란?

Xcode는 Apple이 만든 iOS/macOS 앱 개발 도구입니다. App Store에서 무료로 설치할 수 있습니다.

### Xcode 화면 구조

Xcode를 열면 이런 구조입니다:

```
┌─────────────────────────────────────────────────────────┐
│  ▶ MoaPlus  >  iPhone 16          [빌드 상태 표시]     │  ← 상단 툴바
├──────────┬──────────────────────────┬───────────────────┤
│          │                          │                   │
│ 프로젝트  │                          │  File Inspector   │
│ 네비게이터│      코드 편집 영역        │  (파일 정보)       │
│ (왼쪽)   │      (가운데)             │  (오른쪽)          │
│          │                          │                   │
│ 📁MoaPlus│                        │  Target Membership│
│  📁MoaPlus│                       │  ☑ MoaPlus      │
│  📁MoaPlusKeyboard│                   │  ☑ MoaPlusKeyboard  │
│  📁Tests  │                         │                   │
│          │                          │                   │
├──────────┴──────────────────────────┴───────────────────┤
│                    콘솔 / 디버그 영역 (하단)               │
└─────────────────────────────────────────────────────────┘
```

### "타겟"이란?

이 프로젝트에는 2개의 앱이 들어 있습니다:

| 타겟 이름 | 역할 | 비유 |
|----------|------|------|
| `MoaPlus` | 메인 앱 (설정 화면, 튜토리얼) | 설정을 바꾸는 리모컨 |
| `MoaPlusKeyboard` | 키보드 익스텐션 (실제 키보드) | 실제로 글자를 입력하는 키보드 |

파일을 추가할 때 "이 파일을 어느 앱에서 쓸 건지" 체크하는 것이 **타겟 멤버십**입니다.

## Part 1. Xcode 프로젝트 열기

### Step 0. 프로젝트 열기

1. Finder에서 `moa-plus` 폴더로 이동
2. `MoaPlus.xcodeproj` 파일을 **더블클릭** (파란색 아이콘)
3. Xcode가 열리고 왼쪽에 폴더 구조가 나타남

> 처음 열면 "Indexing..." 메시지가 뜰 수 있습니다. 완료될 때까지 잠시 기다려 주세요.

---

## Part 2. 신규 파일 추가 (Step 1)

코드 파일은 폴더에 이미 있지만, Xcode가 "이 파일이 프로젝트에 포함되어 있다"는 걸 모릅니다.
Xcode에 등록해야 빌드할 때 포함됩니다.

### 1-1. 키보드 파일 추가하기

**Models 폴더 (5개 파일)**

1. Xcode 왼쪽에서 `MoaPlusKeyboard` → `Models` 폴더를 찾아 **우클릭**
2. 메뉴에서 **"Add Files to "MoaPlus"..."** 클릭
3. Finder 창이 열림 → `MoaPlusKeyboard/Models/` 폴더로 이동
4. 아래 5개 파일을 **전부 선택** (첫 번째 클릭 후 `Cmd`를 누른 채로 나머지 클릭):
   - `SwipeProfile.swift`
   - `ColumnGestureOverride.swift`
   - `SecondaryKeyAction.swift`
   - `ShortcutExpansion.swift`
   - `ThemeSettings.swift`
5. 하단의 **"Add to targets:"** 확인:
   ```
   ☑ MoaPlus        ← 체크 (설정 앱에서도 이 모델을 씀)
   ☑ MoaPlusKeyboard    ← 체크 (키보드에서도 이 모델을 씀)
   ```
6. **Add** 클릭

**Engine 폴더 (1개 파일)**

1. 왼쪽에서 `MoaPlusKeyboard` → `Engine` 폴더 **우클릭** → **"Add Files to..."**
2. `AbbreviationEngine.swift` 선택
3. 타겟:
   ```
   ☐ MoaPlus        ← 체크 안 함
   ☑ MoaPlusKeyboard    ← 체크
   ```
4. **Add** 클릭

**Utilities 폴더 (3개 파일)**

1. `MoaPlusKeyboard` → `Utilities` 폴더 **우클릭** → **"Add Files to..."**
2. 아래 3개 파일 선택:
   - `GestureSettings.swift`
   - `HapticManager.swift`
   - `BackgroundImageManager.swift`
3. 타겟:
   - `GestureSettings.swift` → **양쪽 다 체크** (설정 앱에서도 사용)
   - `HapticManager.swift` → **MoaPlusKeyboard만** 체크
   - `BackgroundImageManager.swift` → **MoaPlusKeyboard만** 체크
4. **Add** 클릭

> 팁: 타겟을 잘못 체크했어도 나중에 바꿀 수 있습니다 (1-4 참조).

**ViewModels 폴더 (1개 파일) — 새 폴더**

이 폴더는 Xcode에 아직 없으므로 먼저 만들어야 합니다:

1. 왼쪽에서 `MoaPlusKeyboard` 폴더를 **우클릭**
2. **"New Group"** 클릭
3. 이름을 `ViewModels`로 입력하고 Enter
4. 새로 만든 `ViewModels` 폴더 **우클릭** → **"Add Files to..."**
5. `MoaPlusKeyboard/ViewModels/KeyboardViewModel.swift` 선택
6. 타겟: **MoaPlusKeyboard만** 체크
7. **Add** 클릭

**Views 폴더 (2개 파일)**

1. `MoaPlusKeyboard` → `Views` 폴더 **우클릭** → **"Add Files to..."**
2. 아래 2개 파일 선택:
   - `SpecialCharacterLayerView.swift`
   - `AbbreviationCandidateView.swift`
3. 타겟: **MoaPlusKeyboard만** 체크
4. **Add** 클릭

### 1-2. 설정 앱 파일 추가하기

1. 왼쪽에서 `MoaPlus` 폴더를 **우클릭** → **"New Group"** → 이름: `Settings`
2. 새로 만든 `Settings` 폴더 **우클릭** → **"Add Files to..."**
3. `MoaPlus/Settings/` 폴더의 7개 파일을 **전부 선택**:
   - `SettingsMainView.swift`
   - `InputSettingsView.swift`
   - `SecondaryInputSettingsView.swift`
   - `SpecialCharSettingsView.swift`
   - `AbbreviationSettingsView.swift`
   - `AppearanceSettingsView.swift`
   - `FeedbackSettingsView.swift`
4. 타겟: **MoaPlus만** 체크
5. **Add** 클릭

### 1-3. KeyboardSettings.swift 공유 타겟 확인

이 파일은 이미 프로젝트에 있지만, 양쪽 타겟에서 모두 써야 합니다:

1. 왼쪽에서 `MoaPlusKeyboard` → `Utilities` → `KeyboardSettings.swift` 클릭
2. 오른쪽 패널을 봅니다 (안 보이면 우상단 `⌥⌘1` 또는 오른쪽 패널 버튼 클릭)
3. **"Target Membership"** 섹션을 찾습니다
4. 이렇게 체크:
   ```
   ☑ MoaPlus
   ☑ MoaPlusKeyboard
   ```

같은 방법으로 `SpecialCharacterLayerView.swift`도 양쪽 타겟에 체크합니다.

### 1-4. 타겟 멤버십 나중에 바꾸는 법

실수로 잘못 체크했을 때:

1. 왼쪽에서 해당 **파일 클릭**
2. 키보드 단축키 `Cmd + Option + 1` (오른쪽 패널의 File Inspector 열기)
3. 스크롤을 내려서 **"Target Membership"** 섹션 찾기
4. 체크박스를 원하는 대로 변경

어떤 파일에 뭘 체크해야 하는지 한눈에 보는 표:

## Part 3. App Group 설정 (Step 2)

메인 앱에서 변경한 설정이 키보드에도 적용되려면 "공유 저장소"가 필요합니다.
이것이 App Group입니다.

### 2-1. 프로젝트 설정 화면 찾기

1. Xcode 왼쪽 맨 위에 있는 **프로젝트 파일** (파란색 아이콘, `MoaPlus`)을 클릭
2. 가운데 영역이 프로젝트 설정으로 바뀜
3. 화면 구조:

```
┌──────────────────────────────────────────────┐
│  PROJECT        TARGETS                      │
│  MoaPlus      MoaPlus        ← 이것 클릭  │
│                 MoaPlusKeyboard                 │
│                 MoaPlusKeyboardTests            │
├──────────────────────────────────────────────┤
│  General | Signing & Capabilities | ...      │
│           ↑ 이 탭 클릭                        │
└──────────────────────────────────────────────┘
```

### 2-2. MoaPlus 타겟에 App Group 추가

1. **TARGETS**에서 `MoaPlus` 클릭
2. **"Signing & Capabilities"** 탭 클릭
3. 왼쪽 상단의 **"+ Capability"** 버튼 클릭
4. 검색창에 `App Groups` 입력 → 더블클릭하여 추가
5. App Groups 섹션이 나타남 → **"+"** 버튼 클릭
6. 입력: `group.com.moaki.keyboard` → **OK**

### 2-3. MoaPlusKeyboard 타겟에도 동일하게

1. **TARGETS**에서 `MoaPlusKeyboard` 클릭
2. **"Signing & Capabilities"** 탭 → **"+ Capability"** → `App Groups` 추가
3. **"+"** → `group.com.moaki.keyboard` → **OK**

> 양쪽에 정확히 같은 문자열 `group.com.moaki.keyboard`이 있어야 합니다.

## Part 4. Info.plist 수정 (Step 3)

키보드가 앱과 데이터를 주고받으려면 "전체 접근"을 요청해야 합니다.

### 방법 A: Xcode에서 수정 (추천)

1. 왼쪽에서 `MoaPlusKeyboard` → `Info.plist` 클릭
2. 가운데에 표 형태로 설정이 나타남
3. `RequestsOpenAccess` 항목을 찾음
4. 값을 `NO`에서 **`YES`**로 변경 (드롭다운 클릭)

### 방법 B: 소스 코드로 수정

1. `Info.plist` **우클릭** → **"Open As"** → **"Source Code"**
2. `<false/>`를 `<true/>`로 변경:

```xml
<key>RequestsOpenAccess</key>
<true/>
```

## Part 5. 서명 설정 (Step 4)

Apple은 모든 앱에 "누가 만들었는지" 서명을 요구합니다.

### 4-1. Apple ID 등록 (처음 한 번만)

1. Xcode 메뉴: **Xcode** → **Settings...** (또는 `Cmd + ,`)
2. **Accounts** 탭 클릭
3. 왼쪽 하단 **"+"** 클릭 → **"Apple ID"** 선택
4. 본인의 Apple ID 로그인

### 4-2. 타겟별 서명 설정

1. 프로젝트 파일 클릭 → **TARGETS** → `MoaPlus`
2. **"Signing & Capabilities"** 탭
3. **"Team"** 드롭다운에서 본인 이름(Apple ID) 선택
4. **"Bundle Identifier"**가 고유한지 확인
   - 다른 사람과 겹치면 안 됨
   - 예: `com.myname.MoaPlus`
5. **TARGETS** → `MoaPlusKeyboard`도 동일하게 설정
   - Team: 같은 계정
   - Bundle Identifier: `com.myname.MoaPlus.MoaPlusKeyboard`

> "Automatically manage signing" 체크가 되어 있으면 Xcode가 자동으로 인증서를 관리합니다.

## Part 6. 빌드 확인 (Step 5)

### 5-1. 빌드 대상 선택

Xcode 상단 가운데를 보면:

```
▶  MoaPlus  >  iPhone 16 Pro
     ↑ 스킴        ↑ 기기/시뮬레이터
```

- **스킴**: `MoaPlus` 선택 (드롭다운 클릭)
- **기기**: 시뮬레이터 (`iPhone 16` 등) 또는 연결된 실기기

### 5-2. 빌드 실행

- 키보드: `Cmd + B` (Build)
- 상단에 "Build Succeeded" 표시되면 성공

### 5-3. 빌드 실패 시

상단에 빨간색 에러가 나타나면:

| 에러 메시지 | 의미 | 해결 |
|------------|------|------|
| "Cannot find type 'X' in scope" | 파일이 타겟에 등록 안 됨 | Part 2에서 해당 파일의 타겟 멤버십 확인 |
| "No such module" | import 실패 | 타겟 멤버십 확인 |
| "Provisioning profile" | 서명 문제 | Part 5에서 Team 재설정 |
| "No account for team" | Apple ID 미등록 | Part 5의 4-1 진행 |

에러를 클릭하면 어떤 파일의 몇 번째 줄인지 보여줍니다.

## Part 7. 아이폰에 설치하기

### Step 6. 개발자 모드 켜기 (아이폰에서)

iOS 16부터 개발용 앱을 설치하려면 이 설정이 필요합니다.

1. 아이폰을 **USB 케이블로 Mac에 연결**
2. 아이폰에서 "이 컴퓨터를 신뢰하겠습니까?" → **신뢰** 탭
3. 아이폰: **설정** → **개인정보 보호 및 보안** → 맨 아래로 스크롤 → **개발자 모드**
4. 토글 **켜기** → **재시작** 확인
5. 재부팅 후 → **"개발자 모드를 켜겠습니까?"** → **켜기**

> "개발자 모드" 메뉴가 안 보이는 경우:
> - Xcode 메뉴: **Window** → **Devices and Simulators** 에서 기기를 한 번 인식시킨 후
> - 아이폰 설정을 다시 확인

### Step 7. 앱 설치

1. Xcode 상단에서 빌드 대상을 **연결된 아이폰 이름**으로 변경
   ```
   ▶  MoaPlus  >  [내 아이폰 이름]
   ```
2. **▶ 버튼** 클릭 (또는 `Cmd + R`)
3. 빌드 → 아이폰에 앱 자동 설치 → 자동 실행

### Step 8. 개발자 신뢰 (처음 한 번만)

아이폰에서 "신뢰할 수 없는 개발자" 메시지가 나오면:

1. 아이폰: **설정** → **일반** → **VPN 및 기기 관리**
2. **"개발자 앱"** 섹션에서 본인의 Apple ID 탭
3. **"신뢰"** → **확인**
4. 다시 앱 실행 (`Cmd + R` 또는 아이폰에서 앱 아이콘 탭)

### Step 9. 키보드 추가 (아이폰에서)

1. **설정** → **일반** → **키보드** → **키보드**
2. **새 키보드 추가...** 탭
3. 목록에서 **MoaPlusKeyboard** 선택

### Step 10. 전체 접근 허용 (아이폰에서)

1. **설정** → **일반** → **키보드** → **키보드**
2. 목록에서 **MoaPlusKeyboard** 탭
3. **전체 접근 허용** 토글 → **켜기**
4. 경고 팝업 → **허용**

### Step 11. 키보드 사용

1. **메모** 앱 (또는 아무 텍스트 입력 앱) 열기
2. 텍스트 입력 영역 탭 → 키보드 나타남
3. 키보드 왼쪽 하단 **🌐 버튼**을 **길게 누르기**
4. 목록에서 **MoaPlusKeyboard** 선택
5. 모아키 키보드가 나타나면 완료!

## 빠른 체크리스트

```
Xcode에서:
  [ ] Step 0.  MoaPlus.xcodeproj 열기
  [ ] Step 1.  신규 16개 파일 추가 + 타겟 멤버십 확인
  [ ] Step 2.  App Group 추가 (양쪽 타겟: group.com.moaki.keyboard)
  [ ] Step 3.  Info.plist: RequestsOpenAccess = YES
  [ ] Step 4.  서명 설정 (Apple ID + Team + Bundle ID)
  [ ] Step 5.  Cmd+B 빌드 성공 확인

아이폰에서:
  [ ] Step 6.  개발자 모드 켜기 + 재부팅
  [ ] Step 7.  Cmd+R 로 아이폰에 설치
  [ ] Step 8.  개발자 인증서 신뢰 (처음만)
  [ ] Step 9.  키보드 추가 (설정 → 일반 → 키보드)
  [ ] Step 10. 전체 접근 허용
  [ ] Step 11. 🌐 길게 눌러 키보드 전환
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 키보드 목록에 MoaPlusKeyboard가 없음 | 익스텐션 빌드 실패 | Xcode에서 에러 확인 (빨간 아이콘) |
| 키보드 전환 후 빈 화면 | 코드 로드 실패 | Xcode 하단 콘솔에서 에러 메시지 확인 |
| 설정 변경이 키보드에 반영 안 됨 | App Group 미설정 | Step 2, 3, 10 재확인 |
| "신뢰할 수 없는 개발자" 반복 | 인증서 만료 (무료=7일) | Xcode에서 다시 `Cmd+R` |
| "Provisioning profile" 에러 | 서명 불일치 | Step 4 재설정 |
